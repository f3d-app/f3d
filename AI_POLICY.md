# AI Policy

The F3D maintainers believe that generative AI is a special tool that requires special care. In that regard, if your contributions to F3D are created in whole or in part with generative AI, you MUST:

- **Disclose that you are using AI**
- **Use it in a precise way, to help with specific tasks**
- **Never let it think or talk for you**

Here are different example usages and how acceptable they are:

1. _I use AI to do research for myself._

This is ok as this is your own internal process.

2. _I use AI to do research and share the results with others._

You MUST disclose that the results came from an AI, especially if
no other sources corroborate the results of that research.
The model used should be disclosed as well.
If you do not disclose this, you will be warned and then BANNED on repeated offenses.

3. _I use AI to generate GitHub issues and pull requests, and to communicate and interact with others via chat and discussions._

This is unacceptable and you will be warned and then BANNED on repeated offenses.
The generated content will be deleted when possible. Use your own voice.

4. _I use AI to improve written text in communication channels because English is not my first language._

This is acceptable as long as it is only used as a correction tool and not a writing tool.
Note, however, that this is unnecessary as you will not be judged on the quality of the text.

5. _I use AI to generate example code and then rewrite it from scratch for my own needs when I contribute code._

This is acceptable and doesn't need to be disclosed as long as no AI code is copy-pasted.

6. _I use AI to generate code, test, code comments, documentation or anything that is included in the contributed Git commits._

This must be disclosed in the pull request description, with details of which parts of the code were generated and
which models were used. If you do not, you will be warned and the PR will be closed. You will be BANNED on repeated offenses.

Generated code will be considered to be written by you and under your own copyright. As with any code that you write, the commit author should reflect that.

You are expected to have reviewed and corrected the code thoroughly and should be able to answer any questions about
what the code does and why it was written the way that it was. If not, you will be warned and then BANNED on repeated offenses.

If you are a mentee in the [F3D mentoring program](doc/dev/13-MENTORING.md), you should not use generative AI.
If you do, you will be warned and then kicked from the mentoring program on repeated offenses.

If you are working on a "good-first-issue", you should not use generative AI.
If you do, you will be warned and then the issue will be unassigned on repeated offenses.

7. _I use AI to vibecode my contributions, with no reviews or modifications, or I use agentic AI to generate pull requests and issues._

This is NOT acceptable, if you do so, you will be warned and then BANNED on repeated offenses.
